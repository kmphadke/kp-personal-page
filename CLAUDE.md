# Project Preferences

## Design Preferences

- **Modern, clean designs** - Minimalist aesthetics with clear visual hierarchy
- **Responsive layouts** - All pages must work well on mobile, tablet, and desktop
- **Professional color schemes** - Sophisticated palettes suitable for business/academic contexts
- **Smooth animations** - Subtle transitions and hover effects to enhance user experience

## Code Preferences

- **Well-commented code** - Include explanatory comments for learning purposes
- **Beginner-friendly explanations** - Break down complex concepts in comments
- **Single-file solutions when possible** - Prefer embedded CSS/JS for simpler projects
- **Semantic HTML** - Use appropriate HTML5 elements for accessibility

## Technical Standards

- **Mobile-first approach** - Design for smaller screens first, then scale up
- **Cross-browser compatibility** - Test in major browsers
- **Performance-conscious** - Optimize images, minimize unnecessary code
- **Accessibility** - Include alt text, proper contrast ratios, keyboard navigation

## Preferred Stack

- HTML5, CSS3, JavaScript (vanilla for simple projects)
- Google Fonts for typography
- CSS Grid and Flexbox for layouts
- CSS custom properties (variables) for theming
